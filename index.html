SOX-96456 · Optro AI Review

SOX-96456 options

Six approaches to improving the AI Review panel for SOX/GRC practitioners who already use Optro. The baseline shows the panel as-is; A–F show what's possible.

The shared problem all six attack: an experienced reviewer has to clear a high-volume reconciliation queue — one process change can hit a dozen-plus objects — and today every item costs several expand-read-decide clicks, with no triage step, no completion signal, and a narrow side rail to do it through.
Starting point

The original panel, reskinned to the Luna design system. No UX modifications — this is what all six options compare against.

Original Luna-reskinned panel

  • All content visible immediately — no progressive disclosure

  • Apply, Dismiss, Assign, and Create Issue at equal visual weight
  • List view default — one suggestion per expand cycle

Options A / B / C  ·  New capabilities at three altitudes

Three approaches to improving the AI Review panel, framed as three altitudes of ambition. Each is a working, clickable prototype.

What these three change: the workflow, the interaction model, and ultimately the surface itself — each at a different level of ambition and risk.
● Conservative
OPTION A
Friction reduction inside the current panel
Keep the model, layout, and mental model exactly. Just make each decision cost less.
  • Keyboard-driven review — J/K move, A apply, D dismiss, E edit, S sources
  • Inline word-level redline on every text field
  • Glanceable collapsed containers — field count + confidence
  • Inline top-source line for provenance at a glance
Risk: lowShips fastNo retraining

●● Moderate
OPTION B
Re-sequence the review workflow
Keep the data model, but change how a reviewer moves through volume — from "expand everything" to "triage, then act."
  • Triage pass — recommended disposition + rationale per item
  • One-click sweep to batch-apply low-risk updates
  • Reconciliation progress bar + an explicit finish line
  • Everything in Option A, folded in as table stakes
Risk: mediumBiggest time win

★ Recommended build — B with A's table stakes
●●● Bold
OPTION C
Side panel → reconciliation workspace
When a change hits 14 objects, the side rail is a soda straw. Make reconciliation a full-screen workspace with an impact map.
  • Impact map — the full blast radius, grouped by reason
  • Edit in context with the suggestion as tracked changes
  • Reconciliation as a first-class, auditable session
  • Progress, assignees, keyboard nav across all objects
Risk: highStructural moat

Options D / E / F  ·  UX quality, no new features

Three intensities of restraint — same data model, no added functionality. Pure UX-quality improvement justified by heuristic evaluation, HCAI principles, progressive disclosure, and UX laws.

What these three change: the panel already has the right data. It just presents everything at once, competing for attention. D/E/F reduce that competition — each at a different depth.
● Light
OPTION D
Visual & typographic discipline
No structural change. Make the existing layout read better through color, hierarchy, and spacing alone.
  • Signal badges neutralised — one neutral gray, no color noise
  • Apply is the hero action — Dismiss/Assign/Create Issue retreat
  • Typography & 8-pt rhythm — consistent field label hierarchy
  • Applied/Dismissed states more readable and distinct
Risk: lowestShips fastPure CSS
Open Option D  →
●● Moderate
OPTION E
Re-sequence + progressive disclosure + trust
Same content, different order — what changed first, then why, then the diff on demand, then action.
  • Original text hidden by default — “Show original & reason” reveals context
  • HCAI trust frame — confidence badge + “AI proposes · you decide”
  • Panel chrome compressed into a quieter control row
  • Built on D’s visual base as table stakes
Risk: mediumBiggest scan-speed gain
Open Option E  →
★ Recommended — E on D’s visual base
●●● Deep
OPTION F
Focus-first calm reading model
Focus is the default. One suggestion at a time. HCAI baked in as a template, not an afterthought.
  • Opens in Focus view — one card at a time, prev/next navigation
  • HCAI decision frame on every card — confidence, provenance, you decide
  • Chrome fades to 68% opacity — content/chrome separation
  • Uniform decision unit — same template for every suggestion
Risk: highestBiggest quality leap
